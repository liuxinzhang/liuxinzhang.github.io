发表于 2022-08-25

核心标签：

  1. 排版标签：h（标题）、p（段落）、hr（分割线）、br（换行）
  2. 文本格式化标签：u/ins（下划线）、i/em（倾斜）、s/del（删除线）、b/strong（加粗）
  3. 图片标签：img
    1. 图片属性：src（图片路径）、alt（替换文本）、title（图片悬停标题）
    2. css属性：**text-decoration: none;**去除a标签默认的下划线
  4. 链接标签：a
    1. 链接属性：href（url）、target（跳转方式：_blank-新窗口、_self-默认当前窗口）
  5. 列表标签：ol（order list）、ul（unorder list）、li
    1. ul-css属性：**list-style: none;**去除列表的序标。
  6. 表格标签：table、caption（表格标题）、tr（table row表格行）、th（table head表格头，默认字体加粗+水平居中）、td（table data单元格数据）
    1. table属性：border、cellpadding（单元格内边距）、cellspacing（单元格外边距）、width、height。常用**border="1" cellspacing="0"**来做单条实线表格
    2. th/td属性：rowspan（上下合并/行合并）、colspan（左右合并/列合并）
    3. css属性：**text-align: center;**，文字水平居中，表格文本默认垂直居中，水平左对齐
  7. 表单标签：form、input、button、textarea（多行文本）、select>option（下拉框）、label（称号）
    1. form属性：action（表单提交地址）、method（请求方式，默认get）
    2. input属性：
      1. type：输入框类型
        1. text：默认
        2. password：密码
        3. radio：单选，默认选中checked
        4. checkbox：多选，默认选中checked
        5. file：文件，多文件multiple
      2. name：表单提交字段key，所有需要被提交的数据必须拥有该字段
      3. value：表单提交字段value
      4. placeholder：输入占位提示
      5. readonly：只读
      6. disabled：禁用，被禁用的元素不回被提交
      7. required：必要字段，提交前必须输入内容
      8. maxlength：最大输入长度
      9. pattern：正则校验表达式
    3. button属性：
      1. type：按钮类型
        1. button：普通按钮，未指定时的默认类型
        2. submit：提交，提交按钮和重置按钮必须在表单内才能生效，否则必须通过其他方式提交表单，如form属性或js。
        3. reset：重置
      2. form：按钮在表单外时指定提交表单，form=”表单id”
    4. option属性：
      1. value：表单最终提交值
      2. selected：默认选中
    5. textarea属性：
      1. cols：每行字符数
      2. rows：行数
  8. 无语义化布局标签：div（独占一行）、span（行内标签）

用好线程池肯定需要知道其原理：

工作原理：
线程数小于核心数，来任务就创建新线程，核心数够了，先塞队列，队列满了就创建额外线程，达到最大线程数且队列满了就开始执行拒绝策略。额外线程空闲时间超过保持存活时间就销毁。

ArrayList

发表于 2022-05-01 更新于 2022-08-25 分类于 集合

_ArrayList特点_：

  • 底层实现：采用数组实现（Object[]），支持随机访问（下标访问）
  • 动态扩容：当容量不够用时会进行扩容：变为原来的1.5倍
  • 非线程安全
  • 默认初始化容量是10。
    • JDK7：调用无参构造器时会直接进行默认容量的初始化
    • JDK8：使用了懒加载机制：当添加第一个元素时才开始进行默认容量初始化。有点像单例模式的饿汉式和懒汉式。

事务的四大特性与隔离级别

发表于 2022-02-24 更新于 2022-08-25 分类于 MySQL

_事务_：保证一组数据库操作要么全部成功，要么全部失败。
_四大特性（ACID）_：

  1. 原子性：事务中的数据库操作是一个整体不可分割，要么全部成功要么全部失败。
  2. 一致性：事务在执行前和执行后，数据库的总体状态保持一致。
  3. 隔离性：并发的事务之间不应该相互影响。
  4. 持久性：一旦事务执行成功，对数据库的修改是持久化的，也就是保存到了数据库。
